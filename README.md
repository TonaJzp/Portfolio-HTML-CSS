# Portafolio DIW — Tema 4 (HTML + CSS)

**Autor:** Antonio Jesús Jiménez  
**Asignatura:** Diseño de Interfaces Web (DIW)  
**Entrega:** Tema 4 — Portafolio con HTML y CSS

## Finalidad del trabajo
Crear un portafolio personal utilizando únicamente **HTML y CSS** (sin JavaScript), cumpliendo los requisitos del Tema 4:
- Estructura semántica y código limpio
- Estilos eficientes y coherentes
- Modo claro/oscuro con variables CSS
- Uso de assets (iconos, imágenes y fuentes) organizados en carpeta `assets/`
- Diseño responsive completo (móvil → escritorio → pantallas grandes)
- Transiciones y animaciones (incluyendo 3D y una animación avanzada)
- Formulario de contacto maquetado y centrado

## Despliegue
**URL del sitio:** https://portfolio-html-css-sigma-amber.vercel.app/

## Estructura del proyecto
Este proyecto contiene **un único HTML** y **un único CSS**, además de la carpeta de recursos.

PROYECTO HTML/
├─ index.html
├─ styles.css
├─ README.md
└─ assets/
    ├─ capturasPantalla/
    ├─ fuentes/
    ├─ iconos/
    └─ imagenes/

    
### Contenido de carpetas
- `assets/imagenes/`: imágenes del portfolio (PNG / WEBP / AVIF, incluyendo al menos una con transparencia).
- `assets/iconos/`: iconos en formato SVG.
- `assets/fuentes/`: tipografías usadas en el sitio.
- `assets/capturasPantalla/`: capturas para documentar el proyecto (escritorio y móvil).

## Requisitos técnicos (checklist)
- [ ] Solo 1 archivo HTML (`index.html`) y 1 archivo CSS (`styles.css`).
- [ ] Carpeta `assets/` con imágenes, iconos, fuentes y capturas.
- [ ] Variables CSS para modo claro/oscuro.
- [ ] 2 tipografías (libres de derechos).
- [ ] Al menos 3 iconos SVG.
- [ ] Al menos 3 imágenes en formatos distintos: PNG, WEBP y AVIF (sin JPG ni GIF).
- [ ] Al menos una imagen con transparencia.
- [ ] Al menos una imagen con tamaños adaptativos según pantalla (por ejemplo con `srcset`/`sizes`).
- [ ] Mínimo 3 transiciones CSS distintas.
- [ ] Mínimo 3 animaciones CSS distintas (una con efecto 3D).
- [ ] 1 animación avanzada (scroll / vistas / transición de vistas).
- [ ] Formulario centrado y estilizado.
- [ ] Diseño responsive sin roturas en móvil/tablet/escritorio.
- [ ] Control de versiones: mínimo 1 commit por día en 5 días distintos.
